关于工作之余

发表于 2018-06-14 | 分类于 life

岁月如梭，风雨人生，堪那荆棘载途，怎就一个“累”字概括。

在自己的工作之余，路还好漫长，总想抽出点时间学点啥或者写点自己的所想，所感、所学的东西，苦于没有统一记录，整理，管理，总是搞得很杂乱。最近正好学习上GitHub，觉得还很不错，选择它给自己搭建了此博客。这样可以用来管理下个人的一点学习总结或者一些天方夜谭的想法或是记录自己对生活经历的吐槽。

程序员思维什么梗？

发表于 2018-06-10 | 分类于 life

先给大家讲个两个小故事,故事的发生是这样的。。。。
（一）
小五的老婆想吃水果，最爱吃西瓜和菠萝，她就给小五说：“你今天去街上时，买一个西瓜看见卖菠萝的就买两个！”
小五上街来了，一上街就看见卖菠萝的啦，他就买了两个西瓜，当小五回家后，手里拎着两个大西瓜。
结果小五的老婆见小五回来了，就把小五狠骂了一顿。

（二）
小五家灯坏了，小五就上街买了灯回家，发现没有螺丝刀，就去邻居家借，但是邻居家的螺丝刀有问题，于是就去修理螺丝刀；
修了好久螺丝刀也没修好，天黑了，家里的灯也没修好，小五老婆又狠骂一顿；

两则小故事就是这样啦，也就是这两个梗；
相信也只有程序猿或者学过编程的人员了解了。小故事（一）也就是一个变量match上了，就会去执行某件事。
这就是程序员思维，如果是普通想法的人，会在街上买一个西瓜和两个菠萝带回家，这个就是程序猿的逻辑惯性所造成的。
我也作为一名程序猿，明白其中发生了什么，过程并不重要，重要的是结果，通过小五给老婆买西瓜和菠萝的故事来说，他老婆确实直接的告诉她上街买一个西瓜遇见卖菠萝的就买两个，而小五是见到卖菠萝的啦，就给她买了两个西瓜回家。过程就这样，但结果却不一样。这个也就是程序员思维什么梗？逻辑惯性。

小故事（二）小五修灯却不知变通，把大量的时间耗费在修螺丝刀上，导致灯也没能修好。
在程序猿来说都有过的问题，在解决某个Bug过程中，出现一个无关紧要的小Bug，而耗费大量的时间去解决，最终需要解决的Bug仍然没能解决掉。
程序猿思维什么梗？不能变通的去处理问题，却在核心问题之外徘徊。把时间都耗在了周围优先级不高也没那么重要的事情上，做事儿没能做到轻重缓急。

在此两则有趣小故事，供大家一起分享，两种态度的思考转变。

Linux环境安装Mysql数据库

发表于 2018-06-06 | 分类于 study

详细步骤： 本示例以CentOS6系统安装Mysql 5.6版本为Demo

1. 下载安装介质

首先Mysql 5.6安装包。

2.检查库文件是否存在，如果有则删除

1）执行：
[root@localhost Desktop]$ rpm -qa | grep mysql
mysql-libs-5.1.52-1.el6_0.1.x86_64

删除：
[root@localhost ~]# rpm -e mysql-libs-5.1.52.x86_64 –nodeps
[root@localhost ~]#

2）安装MySql 服务端
rpm -ivh MySQL-server-5.6.22-1.linux_glibc2.5.x86_64.rpm
Preparing… ########################################### [100%]
1:MySQL-server ########################################### [100%]
。
。
。
。
。
开始安装。。。。

3）检测 MySQL 3306 端口是否安打开。测试是否成功可运行 netstat 看 MySQL 端口是否打开，如打开表示服务已经启动，安装成功。MySQL 默认的端口是3306。

执行：
[root@hadoop Mysql]# netstat -nat

4）启动MySql 服务

[root@hadoop Mysql]# service mysql start
Starting MySQL………. SUCCESS!

5） 安装客户端

[root@hadoop Mysql]# rpm -ivh MySQL-client-5.6.22-1.linux_glibc2.5.x86_64.rpm
Preparing… ########################################### [100%]
1:MySQL-client ^C########################################### [100%] 安装完成

6）MySQL的几个重要目录。

MySQL安装完成后不像 SQL Server 默认安装在一个目录，它的数据库文件、配置文件和命令文件分别在不同的目录，了解这些目录非常重要，尤其对于Linux的初学者，因为 Linux本身的目录结构就比较复杂，如果搞不清楚MySQL的安装目录那就无从谈起深入学习。
a、数据库目录 /var/lib/mysql/
b、配置文件 /usr/share/mysql（mysql.server命令及配置文件）
c、相关命令 /usr/bin(mysqladmin mysqldump等命令)
d、启动脚本 /etc/rc.d/init.d/（启动脚本文件mysql的目录）
如：/etc/rc.d/init.d/mysql start/restart/stop/status

7）更改 MySQL 目录

由于MySQL数据库目录占用磁盘比较大，而MySQL默认的数据文件存储目录为 /“var/lib/mysql”，也可以把要把数据目录移到 “/“ 根目录下的 “mysql_data” 目录中（如果做测试用就不用移动了）。

停止 MySql 服务进程

service mysql stop 或者 mysqladmin -u root -p shutdown

MySQL默认用户名为”root”，此处的”root”与Linux的最高权限用户”root”不是一同一个，而且默认的用户”root”的密码为空，所以上图中让输入密码，直接点击回车即可。

把 “/var/lib/mysql” 整个目录移到 “/mysql_data”
mv /var/lib/mysql /mysql_data

找到my.cnf配置文件

如果”/etc/“目录下没有my.cnf配置文件，请到 “/usr/share/mysql/“ 下找到 my-default.cnf 文件，拷贝其中一个合适的配置文件到 “/etc/“ 并改名为 “my.cnf” 中。命令如下：
cp /usr/share/mysql/my-medium.cnf　 /etc/my.cnf

编辑MySQL的配置文件

“/etc/my.cnf” 为保证MySQL能够正常工作，需要指明”mysql.sock”文件的产生位置，以及默认编码修改为UTF-8。用下面命令：
vim /etc /my.cnf
[mysqld] #下添加以下命令
socket = /var/lib/mysql/mysql/mysql.sock
datadir=/mysql_data/mysql #如果你的数据存储位置要改变的话需要在这里指定存储目录
character-set-server=utf8
lower_case_table_names=1

（注意linux下mysql安装完后是默认：区分表名的大小写，不区分列名的大小写；# lower_case_table_names = 0 0：区分大小写，1：不区分大小写）
注意：
1）最后一行，我在本地安装的时候没有添加此设置也可以照样运行起来，加不加还要看你们自己了。

2）还有在此配置文件还有 [client] 的一下配置 其中也可以设置 socket ，datadir 两个选项，我这里也没有（此处不做配置，因为只是安装 MySql 测试）

3）如果你修改了数据的存储位置目录 ，则可以执行以下命令来修改MySQL启动脚本 “/etc/rc.d/init.d/mysql”
最后，需要修改MySQL启动脚本 /etc/rc.d/init.d/mysql，修改 datadir=/mysql_data/mysql。
vim /etc/rc.d/init.d/mysql

8）重新启动MySQL服务

service mysql start
发生错误：Starting MySQL ERROR! The server quit without updataing PID file….
解决方法是：
vi /etc/selinux/config
把 SELINUX=enforcing 改为 SELINUX=disabled 后存盘退出重启机器试试，必须要重启，很关键。

MySQL 没有初始化表

/usr/bin/mysql_install_db –user=mysql

启动MySQL 服务：

service mysql start
service mysql start 就可以重启MySQL 服务了。

9）修改登录密码

MySQL默认没有密码，安装完修密码，直接登录
检查 MySql 端口是否开启：
[root@Hadoop ~]#netstat -ntlp | grep 3306
tcp 0 0 :::3306 :::* LISTEN 5052/mysqld

$：mysql
报错：ERROR 2002 (HY000): Can’t connect to local MySQL server through socket ‘/var/lib/mysql/mysql.sock’ (2)
发现 与设置 MySql 服务器的 socket 位置不一样。改一下 /etc/my.cn
修改
socket = /var/lib/mysql/mysql.sock –保存；

再次进入：
[root@hadoop init.d]# mysql
又报错。。。
ERROR 1045 (28000): Access denied for user ‘root‘@’localhost’ (using password: NO)

解决方法：
[root@hadoop ~]#mysqld_safe –user=mysql –skip-grant-tables –skip-networking & mysql -u root mysql

再次进入，执行：
[root@hadoop ~]# mysql
Welcome to the MySQL monitor. Commands end with ; or \g.
Your MySQL connection id is 2
Server version: 5.6.22 MySQL Community Server (GPL)

Copyright (c) 2000, 2014, Oracle and/or its affiliates. All rights reserved.

Oracle is a registered trademark of Oracle Corporation and/or its
affiliates. Other names may be trademarks of their respective
owners.

Type ‘help;’ or ‘\h’ for help. Type ‘\c’ to clear the current input statement.
mysql>
OK 安装成功了。。。

在没有添加密码前，直接输入”mysql”就能登录到MySQL数据库里。 修改登录密码：
[root@hadoop ~]mysqladmin -u root password ‘root’
格式：mysqladmin -u用户名 -p旧密码 password 新密码

如果已经连接上了 MySql 可以使用下面语句 ：
mysql> use mysql;
mysql> UPDATE user SET Password=PASSWORD(‘root’) where USER=’root’;
Query OK, 4 rows affected (0.04 sec)
Rows matched: 4 Changed: 4 Warnings: 0
修改成功；

Oracle数据库备份与还原

发表于 2018-06-06 | 分类于 study

1.备份

注：本次演示在C盘，首先在C盘新建backup文件夹，新建完成后，
在SQl Plus下执行：
create directory dump_dir as ‘C:\backup’;
GRANT READ,WRITE ON DIRECTORY dump_dir to admin;
命令行执行
expdp SCHEMAS= admin directory=dump_dir dumpfile=admin1231.dmp logfile=admin_1231.log
或
exp Admin/Admin#@ORC file=C:/backup/admin1231.dmp log=C:/backup/admin_1231.log owner=Admin

注：Admin/Admin#@ORC 分别为用户名/密码/数据库名

2.还原

select ‘drop view ‘||view_name||’;’ from user_views;
select ‘drop table ‘||table_name||’;’ from user_tables;
注：以上每执行一条将执行后的结果复制到SQL文本中执行，删除所有的表与视图

开始还原：
impdp SCHEMAS= admin directory=DATA_PUMP_DIR dumpfile=admin1231.dmp table_exists_action=replace logfile=admin_1231.log

Linux环境安装Oracle数据库

发表于 2018-06-06 | 分类于 study

详细步骤： 本示例以CentOS6系统安装Oracle 11g为Demo

1. 下载安装介质

首先在Oracle的网站上下载最新的Oracle 11.2安装包，因为CentOS6装的是x86_64的，所以，Oracle也选择Linux x86_64的安装包。下载下来是2个zip文件包

2. 安装Oracle需要的软件依赖包

yum -y install binutils “compat-libstdc++” elfutils-libelf elfutils-libelf-devel expat gcc gcc-c++ glibc glibc-common glibc-devel glibc-headers libaio libaio-devel libgcc libstdc++ libstdc++ make “pdksh” sysstat unixODBC unixODBC-devel
yum -y install compat-libstdc++-33.x86_64 ksh.x86_64
yum -y install libaio.i686 glibc.i686 compat-libstdc++-33.i686 libaio-devel.i686 libgcc.i686 libstdc++.i686 unixODBC.i686 unixODBC-devel.i686

注：若不支持yum安装依赖包，则需要手动下载所有需要的依赖包手动安装，安装执行： rpm -ivh 依赖包名

用下面命令，看看是否安装了所有的包，
rpm -q binutils compat-libstdc elfutils-libelf elfutils-libelf-devel elfutils-libelf-devel-static gcc gcc-c++ glibc glibc-common glibc-devel glibc-headers kernel-headers ksh libaio libaio-devel libgcc libgomp libstdc++ libstdc++-devel make sysstat

3.创建oracle用户

创建相关用户和组，进入到 /usr/sbin 目录下分别执行：
step 1： groupadd oinstall 
step 2： groupadd dba 
step 3： useradd -g oinstall -G dba oracle 
step 4： passwd oracle //设置oracle账户密码

4.更改内核参数

在/etc/sysctl.conf后面增加如下参数：

fs.aio-max-nr = 1048576 
fs.file-max = 6815744 
kernel.shmall = 2097152 
kernel.shmmax = 2147483648 
kernel.shmmni = 4096 
kernel.sem = 250 32000 100 128 
net.ipv4.ip_local_port_range = 9000 65500 
net.core.rmem_default = 262144 
net.core.rmem_max = 4194304 
net.core.wmem_default = 262144 
net.core.wmem_max = 1048586

/sbin/sysctl -p 确认更改

5.更改Oracle用户的shell限制

在/etc/security/limits.conf后面增加如下行
oracle soft nproc 2047 
oracle hard nproc 16384 
oracle soft nofile 1024 
oracle hard nofile 65536

在/etc/pam.d/login 后面增加如下行
session required pam_limits.so

在 /etc/profile后面增加如下行
if [ $USER = "oracle" ]; then 
if [ $SHELL = "/bin/ksh" ]; then 
ulimit -p 16384 
ulimit -n 65536 
else 
ulimit -u 16384 -n 65536 
fi 
fi

如果使用C-shell,则更改 /etc/csh.login
if ( $USER == “oracle” ) then
limit maxproc 16384
limit descriptors 65536
endif

6.创建Oracle安装目录

我的oracle安装在了/opt/oracle/
Inventory目录在 /opt/oraInventory/

执行以下语句：
mkdir -p /opt/oracle/
chown -R oracle:oinstall /opt/oracle/
chmod -R 775 /opt/oracle/

mkdir -p /opt/oraInventory/
chown -R oracle:oinstall /opt/oraInventory/
chmod -R 775 /opt/oraInventory/

7.更改Oracle用户的profil

以Oracle用户登录，su oracle
运行
vi .bash_profile
增加下列行
umask 022
ORACLE_BASE=/opt/oracle
ORACLE_SID=orcl
export ORACLE_BASE ORACLE_SID

unset ORACLE_HOME
unset TNS_ADMIN
更改完成后，退出oracle用户，重新登录，用以下命令检查是否更改成功
umask
env | more

至此，环境配置完毕，可以开始安装Oracle了，选择先安装软件，再安装数据库

8.开始安装Oracle软件

step 1 :用新建的oracle用户登录
Step 2 :确保oracle用户对安装的软件包有足够权限
step 3：解压上传的安装包
Step 4：解压后，会生成一个database文件夹，进入解压后的database目录下查看安装介质

注：oracle安装需要图像化界面操作，前提需安装好Xmanager工具，以供图形化界面展示操作
在安装好Xmanager工具后，linux环境需配置好
进入到 /home/oracle 目录，执行：vi .bash_profile 编辑配置文件
增加：export DISPLAY=192.168.16.124:0.0
注：192.168.16.124是安装Xmanager工具的物理主机IP地址，后面的0.0是Xmanager工具图形界面认证的标识

开始安装：

step 1:先打开安装的Xmanager ：双击执行Xmanager下的Xmanager -Passive 如图：


step 2:进入到刚解压的database目录下，执行：./runInstaller 开始安装，Xmanager会展示出图形界面
